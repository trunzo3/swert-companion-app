# Replit Prompt 8 — Section Refinements and Content Variant Additions

## Context

This prompt covers 8 changes to section components, seed behavior, and content variant support. Do NOT touch the admin panel layout, login flow, sidebar, home screen, or any infrastructure. These are all participant-facing section refinements and one backend seed change.

CRITICAL: The seed script must NOT reset existing cohorts. See item 1.

---

## 1. Seed Script Protection

The seed script currently calls resetCohortSectionsToDefaults for existing cohorts, which overwrites any custom display names, codes, ordering, and visibility settings the admin has configured.

Fix: Change the seed behavior so that resetCohortSectionsToDefaults is ONLY called for newly created cohorts (when the seed creates a default cohort because none exist). For existing cohorts, the seed should only ADD new sections that don't already exist in cohort_sections — it should never delete, reorder, or modify existing cohort_sections rows.

Logic:
- If a section ID exists in ALL_SECTIONS but NOT in a cohort's cohort_sections: insert it with default values at the end of its level
- If a section ID exists in both ALL_SECTIONS and cohort_sections: do not touch it
- If a section ID exists in cohort_sections but NOT in ALL_SECTIONS: leave it alone (it may be a generic section)
- Never delete cohort_sections rows during seed
- Never overwrite display_name, code, sort_order, visible, or code_active for existing rows

## 2. Draft with RICECO — Restyle Drafting Fields

The "Draft with RICECO" section has a "DRAFTING WORKSPACE" with six plain text fields (R: ROLE, I: INSTRUCTION, etc.) that look like basic form inputs. Restyle them to match the visual treatment of the RICECO Framework teaching section.

For each of the six RICECO fields, render:

One line containing: gold circle (40px, background #C8963E, white letter inside, border-radius 50%) + bold label + helper subtitle in muted text. All on the same line, vertically centered.

Directly below that line: the textarea input field with auto-save.

The six fields with their labels and helper text:

- R circle | Role — Who is the AI acting as? | [textarea]
- I circle | Instruction — What exactly do you want it to do? | [textarea]
- C circle | Context — What background information is needed? | [textarea]
- E circle | Examples — What does good look like? | [textarea]
- C circle | Constraints — What rules must it follow? | [textarea]
- O circle | Output — How should the final result be formatted? | [textarea]

The label (e.g., "Role") should be bold, 15px, navy. The helper text (e.g., "Who is the AI acting as?") should be regular weight, 13px, muted gray color, separated from the label by an em dash or a pipe character.

Keep the "DRAFTING WORKSPACE" header above all six fields.

### Add "Copy Full Prompt" Button

Below the sixth textarea (Output), add a prominent button: "Copy Full Prompt"

When clicked, concatenate all six fields into a single formatted prompt string:

Role: [content of R field]
Instructions: [content of I field]
Context: [content of C field]
Examples: [content of E field]
Constraints: [content of C field]
Output Format: [content of O field]

Copy the concatenated string to the clipboard. Show "Copied!" feedback (same pattern as the CopyButton component used elsewhere). Skip any empty fields from the concatenation (don't include "Role: " with nothing after it).

Style the button: navy background, white text, 14px font-weight 600, padding 12px 32px, border-radius 8px. Centered below the fields. Use a clipboard icon next to the text.
